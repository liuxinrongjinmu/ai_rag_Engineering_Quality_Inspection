## 项目概述

### 背景

公路工程质量检测涉及大量规范文档（JTG、GB等标准），检测人员在实际工作中需要频繁查阅这些文档以确定检测项目、频率、方法等。传统查阅方式效率低下，且容易遗漏关键信息。

### 目标

构建一个智能问答系统，能够：
1. 基于本地规范文档库进行精准检索
2. 将口语化问题转换为专业术语查询
3. 提供可追溯的答案来源
4. 本地无答案时自动联网搜索权威来源

### 核心特性

| 特性 | 说明 |
|------|------|
| 混合检索 | 向量检索 + BM25关键词检索并行执行，融合两种检索优势 |
| 查询重写 | 口语化问题自动转换为规范术语 |
| 来源追溯 | 每个答案可追溯到具体文档、页码、章节 |
| 网络补充 | 本地无答案时自动联网搜索权威来源 |
| 智能缓存 | 常见问题缓存，命中时<100ms响应 |
| 流式输出 | 支持SSE流式返回，用户感知更快 |

## 系统架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                              用户查询                                    │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                         Query Orchestrator                               │
│                         (查询编排器)                                      │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                    ┌───────────────┼───────────────┐
                    ▼               ▼               ▼
            ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
            │ Query Rewriter│ │ Hybrid       │ │ Web Retriever│
            │ (查询重写)     │ │ Retriever    │ │ (网络检索)    │
            └──────────────┘ │ (混合检索)    │ └──────────────┘
                             └──────┬───────┘
                                    │
                    ┌───────────────┼───────────────┐
                    ▼               ▼               ▼
            ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
            │ Milvus       │ │ BM25 Retriever│ │ Reranker     │
            │ (向量检索)    │ │ (关键词检索)  │ │ (重排序)      │
            └──────────────┘ └──────────────┘ └──────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                            RAG Engine                                    │
│                         (答案生成引擎)                                    │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐         │
│  │ Context Builder │  │ LLM Generator   │  │ Quality Evaluator│         │
│  │ (上下文构建)     │  │ (答案生成)      │  │ (质量评估)       │         │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘         │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                              API Response                                │
│                    { answer, sources, query_time_ms }                   │
└─────────────────────────────────────────────────────────────────────────┘
```

### 数据流程

1. **查询接收** → 用户提交问题
2. **缓存检查** → 检查是否有缓存的答案
3. **查询重写** → 口语化问题转换为规范术语
4. **混合检索** → 向量检索 + BM25检索并行执行
5. **结果融合** → 加权融合两种检索结果
6. **网络补充** → 本地结果不足时触发网络检索
7. **重排序** → 对所有结果进行加权排序
8. **答案生成** → LLM基于检索结果生成答案
9. **缓存存储** → 缓存答案供后续使用

## 核心功能模块

### 1. 查询重写器 (Query Rewriter)

将用户口语化查询转换为规范术语，提升检索准确率。

```python
# 示例
"压实度怎么检" → "压实度检测方法"
"多久检一次" → "检测频率"
"粉煤灰检什么" → "粉煤灰检测项目"
```

**实现方式：**
- 规则映射：预定义口语→术语映射表
- LLM增强：使用Qwen-Turbo进行智能重写

### 2. 混合检索器 (Hybrid Retriever)

融合向量检索和BM25检索的优势，**并行执行**提升效率：

| 检索方式 | 权重 | 优势 | 劣势 |
|----------|------|------|------|
| 向量检索 | 60% | 语义理解能力强 | 专业术语可能不准 |
| BM25检索 | 40% | 精确匹配关键词 | 无法理解语义 |

**融合策略：**
- 并行执行：向量检索和BM25检索同时进行，减少约0.5-1秒
- 加权融合：`score = vector_score * 0.6 + bm25_score * 0.4`
- 双重命中奖励：两种检索都命中的结果额外加成20%

### 3. 文本切片器 (Text Chunker)

采用固定大小分块策略，优化表格处理：

```python
# 切片参数
CHUNK_SIZE = 1000      # 切片大小（字符）
CHUNK_OVERLAP = 200    # 重叠区域
MIN_CHUNK_SIZE = 100   # 最小切片
```

**特殊处理：**
- 表格内容独立切片，保持完整性
- 按段落优先切分
- 大段落按句子切分

### 4. RAG引擎 (RAG Engine)

基于检索结果生成答案：

```python
# 答案生成流程
1. 构建上下文（最多6000字符）
2. 调用Qwen-Plus生成答案
3. 内部评估答案质量（不返回用户）
```

**Prompt设计：**
- 角色设定：公路工程质量检测专家
- 回答原则：准确性优先、完整提取、标注来源
- 格式要求：先给答案，再列详情，最后标注来源

### 5. 网络检索器 (Web Retriever)

本地无答案时自动联网搜索：

**触发条件：** 本地检索最高分 < 0.5

**权威域名白名单：**
- mot.gov.cn（交通运输部）
- mohurd.gov.cn（住建部）
- std.samr.gov.cn（国家标准委）
- openstd.samr.gov.cn（国家标准公开）

### 6. 智能缓存 (Query Cache)

内存缓存常见问题的答案，大幅提升响应速度：

**缓存策略：**
- 最大缓存数量：1000条
- 缓存过期时间：1小时
- 命中时响应时间：<100ms

**缓存键生成：**
- 基于问题内容 + 是否使用网络检索
- MD5哈希确保唯一性

### 7. 流式输出 (Streaming Output)

支持SSE (Server-Sent Events) 流式返回答案：

**优势：**
- 用户实时看到答案生成过程
- 感知延迟大幅降低
- 不影响答案质量

**接口：** `POST /api/v1/query/stream`

## 快速开始

### 1. 环境要求

- Python 3.10+
- Docker & Docker Compose（用于Milvus）
- Windows/Linux/macOS

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动Milvus

```bash
# 启动Milvus服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 等待Milvus完全启动（约30秒）
```

### 4. 配置环境变量

```bash
# 复制配置模板
cp .env.example .env

# 编辑.env文件，填入API Key
```

**必需配置：**

```env
# 阿里云DashScope API Key（用于Embedding和LLM）
DASHSCOPE_API_KEY=sk-xxxxxxxx

# Tavily API Key（用于网络检索）
TAVILY_API_KEY=tvly-xxxxxxxx

# Milvus配置
MILVUS_HOST=localhost
MILVUS_PORT=19530
```

**获取API Key：**
- DashScope: https://dashscope.console.aliyun.com/
- Tavily: https://tavily.com/

### 5. 数据入库

```bash
python scripts/ingest.py
```

**入库流程：**
1. 扫描 `data/processed/` 目录下的文件
2. 解析Markdown和Excel文件
3. 文本切片（1000字符/块）
4. 向量化并写入Milvus
5. 构建BM25索引

### 6. 启动服务

```bash
# 方式一：直接启动
python -m uvicorn app.main:app --host 127.0.0.1 --port 5001

# 方式二：开发模式（热重载）
python -m uvicorn app.main:app --host 127.0.0.1 --port 5001 --reload
```

### 7. 访问API文档

打开浏览器访问：**http://localhost:5001/docs**

### 8. 测试系统

```bash
# 交互式测试
python test_api.py

# 选择模式：
# 1 - 运行预设测试场景
# 2 - 交互模式（输入自己的问题）
# 3 - 健康检查
```

```json
{
    "code": 0,
```

## 问答结果控制

### 响应数据结构

API返回的问答结果由以下字段组成：

```python
# 定义位置: app/models/response.py

class QueryData(BaseModel):
    answer: str                    # LLM生成的答案
    sources: List[SourceInfo]      # 来源信息列表
    query_time_ms: int             # 查询耗时(毫秒)
    used_web_search: bool          # 是否使用了网络检索

class SourceInfo(BaseModel):
    doc_id: str                    # 文档ID
    doc_name: str                  # 文档名称
    page: Optional[int]            # 页码
    section: Optional[str]         # 章节
    content: str                   # 原文内容（截取前500字符）
    source_type: SourceType        # 来源类型: local/web
    url: Optional[str]             # URL（仅网络来源）
```

### 数据流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户请求                                  │
│  POST /api/v1/query {question, options}                         │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    query.py (API层)                             │

└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                 orchestrator.py (编排器)                         │
│                                                                 │
│  1. 缓存检查 ──→ 命中则直接返回                                   │
│  2. hybrid_retriever.retrieve() ──→ 检索结果                     │
│  3. rag_engine.generate() ──→ 生成答案                           │
│  4. rag_engine.extract_sources() ──→ 提取来源                    │
```
