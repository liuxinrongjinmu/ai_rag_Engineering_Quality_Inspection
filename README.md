## 项目概述

### 背景

公路工程质量检测涉及大量规范文档（JTG、GB等标准），检测人员在实际工作中需要频繁查阅这些文档以确定检测项目、频率、方法等。传统查阅方式效率低下，且容易遗漏关键信息。

### 目标

构建一个智能问答系统，能够：
1. 基于本地规范文档库进行精准检索
2. 将口语化问题转换为专业术语查询
3. 提供可追溯的答案来源
4. 本地无答案时自动联网搜索权威来源

### 核心特性

| 特性 | 说明 |
|------|------|
| 混合检索 | 向量检索 + BM25关键词检索，融合两种检索优势 |
| 查询重写 | 口语化问题自动转换为规范术语 |
| 来源追溯 | 每个答案可追溯到具体文档、页码、章节 |
| 网络补充 | 本地无答案时自动联网搜索权威来源 |
| LLM评估 | 内部评估答案质量，持续优化系统 |

### 数据流程

1. **查询接收** → 用户提交问题
2. **查询重写** → 口语化问题转换为规范术语
3. **混合检索** → 向量检索 + BM25检索并行执行
4. **结果融合** → 加权融合两种检索结果
5. **网络补充** → 本地结果不足时触发网络检索
6. **重排序** → 对所有结果进行加权排序
7. **答案生成** → LLM基于检索结果生成答案
8. **质量评估** → 内部评估答案质量（不返回用户）

## 核心功能模块

### 1. 查询重写器 (Query Rewriter)

将用户口语化查询转换为规范术语，提升检索准确率。

```python
# 示例
"压实度怎么检" → "压实度检测方法"
"多久检一次" → "检测频率"
"粉煤灰检什么" → "粉煤灰检测项目"
```

**实现方式：**
- 规则映射：预定义口语→术语映射表
- LLM增强：使用Qwen-Turbo进行智能重写

### 2. 混合检索器 (Hybrid Retriever)

融合向量检索和BM25检索的优势：

| 检索方式 | 权重 | 优势 | 劣势 |
|----------|------|------|------|
| 向量检索 | 60% | 语义理解能力强 | 专业术语可能不准 |
| BM25检索 | 40% | 精确匹配关键词 | 无法理解语义 |

**融合策略：**
- 加权融合：`score = vector_score * 0.6 + bm25_score * 0.4`
- 双重命中奖励：两种检索都命中的结果额外加成20%

### 3. 文本切片器 (Text Chunker)

采用固定大小分块策略，优化表格处理：

```python
# 切片参数
CHUNK_SIZE = 1000      # 切片大小（字符）
CHUNK_OVERLAP = 200    # 重叠区域
MIN_CHUNK_SIZE = 100   # 最小切片
```

**特殊处理：**
- 表格内容独立切片，保持完整性
- 按段落优先切分
- 大段落按句子切分

### 4. RAG引擎 (RAG Engine)

基于检索结果生成答案：

```python
# 答案生成流程
1. 构建上下文（最多6000字符）
2. 调用Qwen-Plus生成答案
3. 内部评估答案质量（不返回用户）
```

**Prompt设计：**
- 角色设定：公路工程质量检测专家
- 回答原则：准确性优先、完整提取、标注来源
- 格式要求：先给答案，再列详情，最后标注来源

### 5. 网络检索器 (Web Retriever)

本地无答案时自动联网搜索：

**触发条件：** 本地检索最高分 < 0.5

**权威域名白名单：**
- mot.gov.cn（交通运输部）
- mohurd.gov.cn（住建部）
- std.samr.gov.cn（国家标准委）
- openstd.samr.gov.cn（国家标准公开）

## 快速开始

### 1. 环境要求

- Python 3.10+
- Windows/Linux/macOS

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 配置环境变量

```bash
# 复制配置模板
cp .env.example .env

# 编辑.env文件，填入API Key
```

**必需配置：**

```env
# 阿里云DashScope API Key（用于Embedding和LLM）
DASHSCOPE_API_KEY=sk-xxxxxxxx

# Tavily API Key（用于网络检索）
TAVILY_API_KEY=tvly-xxxxxxxx
```

**获取API Key：**
- DashScope: https://dashscope.console.aliyun.com/
- Tavily: https://tavily.com/

### 4. 数据入库

```bash
python scripts/ingest.py
```

**入库流程：**
1. 扫描 `data/processed/` 目录下的文件
2. 解析Markdown和Excel文件
3. 文本切片（1000字符/块）
4. 向量化并写入ChromaDB
5. 构建BM25索引

### 5. 启动服务

```bash
# 方式一：直接启动
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

# 方式二：开发模式（热重载）
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 6. 访问API文档

打开浏览器访问：**http://localhost:8000/docs**

### 7. 测试系统

```bash
# 交互式测试
python test_api.py

# 选择模式：
# 1 - 运行预设测试场景
# 2 - 交互模式（输入自己的问题）
# 3 - 健康检查
```
