# 工程质检RAG系统 - 前端需求文档

## 一、项目背景

**项目名称**：工程质检RAG系统前端

**核心目标用户**：

- 甲方监理（关注合规性、责任界定）
- 总工程师（关注规范依据、条文解释）
- 现场质检员（关注快速答案、操作指引）
- 企事业单位（关注标准查询、报告编制）

**核心场景**：用户输入工程质量检测相关问题，系统返回准确答案并展示来源依据。

## 三、功能模块

### 模块1：问答对话区

**功能描述**：

- 展示用户与系统的对话历史（类似ChatGPT的对话界面）
- 用户消息靠右显示，系统回复靠左显示
- 支持滚动查看历史记录
- 新消息自动滚动到底部

**系统回复内容**：

1. **答案文本**：系统生成的答案
2. **来源卡片**：
   - 文档名称（如：JTG F80-1-2017）
   - 页码/章节（如：第15页 4.2.2节）
   - 来源类型标签（本地知识库/网络检索）
   - 「查看原文」按钮
3. **置信度指示**：显示答案的可信程度（如：置信度 92%）
4. **响应时间**：显示查询耗时（如：1.2秒）

### 模块2：输入区域

**功能描述**：

- 文本输入框（支持多行输入）
- 发送按钮（点击或回车发送）
- 输入提示文字
- 快捷问题推荐（可选）

**交互要求**：

- 输入框为空时，发送按钮禁用
- 发送后清空输入框
- 发送过程中显示加载状态

### 模块3：来源详情弹窗

**触发方式**：点击「查看原文」按钮

**弹窗内容**：

```
┌─────────────────────────────────────────────┐
│  来源详情                              [×]  │
├─────────────────────────────────────────────┤
│  文档: JTG F80-1-2017 公路工程质量检验...    │
│  页码: 第15页                               │
│  章节: 4.2.2 土方路基                        │
├─────────────────────────────────────────────┤
│  原文内容:                                   │
│  ┌─────────────────────────────────────┐   │
│  │ 土方路基压实度检测频率...             │   │
│  │ （完整段落内容）                      │   │
│  └─────────────────────────────────────┘   │
│                                             │
│  [复制内容]                                 │
└─────────────────────────────────────────────┘
```

### 模块4：系统状态指示（可选）

**位置**：页面底部或顶部

**内容**：

- 系统状态：正常/异常
- 知识库统计：已入库文档数、切片数

## 四、交互流程

### 主流程

```
1. 用户在输入框输入问题
2. 点击发送或按回车
3. 输入框清空，用户消息显示在对话区
4. 显示加载动画（系统思考中...）
5. 系统回复显示在对话区
6. 用户可点击「查看原文」查看来源详情
7. 用户可继续提问，对话历史保留
```

### 异常流程

```
- 网络错误：显示"网络连接失败，请重试"
- 系统繁忙：显示"系统繁忙，请稍后再试"
- 无结果：显示"未找到相关信息，建议换种方式提问"
```

## 五、接口对接

### 1. 问答接口

**请求**：

```
POST /api/v1/query
Content-Type: application/json
```

**请求体**：

```json
{
    "question": "土方路基压实度检测频率是多少？",
    "options": {
        "use_web_search": true,
        "top_k": 5
    }
}
```

**响应体**：

```json
{
    "code": 0,
    "message": "success",
    "data": {
        "answer": "根据JTG F80-1-2017《公路工程质量检验评定标准》...",
        "sources": [
            {
                "doc_id": "abc123",
                "doc_name": "JTG F80-1-2017 公路工程质量检验评定标准",
                "page": 15,
                "section": "4.2.2",
                "content": "土方路基压实度检测频率为...",
                "source_type": "local",
                "url": null
            }
        ],
        "confidence": 0.92,
        "query_time_ms": 1234,
        "used_web_search": false
    }
}
```

**字段说明**：

| 字段                      | 类型     | 说明              |
| ----------------------- | ------ | --------------- |
| answer                  | string | 系统生成的答案         |
| sources                 | array  | 来源列表            |
| sources\[].doc\_id      | string | 文档ID（用于查询详情）    |
| sources\[].doc\_name    | string | 文档名称            |
| sources\[].page         | int    | 页码              |
| sources\[].section      | string | 章节号             |
| sources\[].content      | string | 原文片段            |
| sources\[].source\_type | string | 来源类型：local/web  |
| sources\[].url          | string | 网络来源URL（仅web类型） |
| confidence              | float  | 置信度（0-1）        |
| query\_time\_ms         | int    | 查询耗时（毫秒）        |
| used\_web\_search       | bool   | 是否使用了网络检索       |

### 2. 来源详情接口

**请求**：

```
GET /api/v1/source/{chunk_id}
```

**响应体**：

```json
{
    "code": 0,
    "message": "success",
    "data": {
        "chunk_id": "abc123",
        "doc_id": "def456",
        "doc_name": "JTG F80-1-2017 公路工程质量检验评定标准",

        "full_content": "完整的原文内容...",
        "context_before": "前文上下文...",
        "context_after": "后文上下文..."
    }
}
```

## 六、UI/UX要求

### 视觉风格

- **配色**：专业、稳重（建议蓝灰色系，符合工程行业特点）
- **字体**：清晰易读（中文建议使用系统默认或思源黑体）
- **布局**：简洁大方，重点突出问答内容

### 响应式设计

| 设备  | 布局要求            |
| --- | --------------- |
| 桌面端 | 对话区居中，最大宽度800px |
| 平板端 | 对话区自适应宽度        |
| 移动端 | 全屏对话区，输入框固定底部   |

### 加载状态

- 发送问题后显示"思考中..."动画
- 来源卡片加载时显示骨架屏

### 来源类型标识

| 来源类型  | 标签颜色 | 标签文字         |
| ----- | ---- | ------------ |
| 本地知识库 | 绿色   | "规范文档"       |
| 网络检索  | 橙色   | "网络来源（仅供参考）" |

***

## 七、非功能需求

| 指标    | 要求                               |
| ----- | -------------------------------- |
| 首屏加载  | < 2秒                             |
| 响应感知  | 用户发送后立即显示消息，不等待接口返回              |
| 错误处理  | 友好提示，不显示技术错误信息                   |
| 浏览器兼容 | Chrome、Edge、Firefox、Safari最新两个版本 |
| 移动端适配 | iOS Safari、Android Chrome        |

## 八、技术建议

### 推荐技术栈

| 类别      | 推荐方案                      |
| ------- | ------------------------- |
| 框架      | React 18 / Vue 3          |
| UI组件库   | Ant Design / Element Plus |
| HTTP客户端 | Axios                     |
| 状态管理    | Zustand / Pinia           |
| 构建工具    | Vite                      |

### API基础配置

```javascript
const API_BASE_URL = 'http://localhost:8000';

const apiClient = axios.create({
  baseURL: API_BASE_URL,
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json'
  }
});
```

### 调用示例

```javascript
// 发送问题
async function sendQuestion(question) {
  const response = await apiClient.post('/api/v1/query', {
    question: question,
    options: {
      use_web_search: true,
      top_k: 5
    }
  });
  return response.data;
}

// 获取来源详情
async function getSourceDetail(chunkId) {
  const response = await apiClient.get(`/api/v1/source/${chunkId}`);
  return response.data;
}
```

## 九、开发注意事项

1. **跨域问题**：后端已配置CORS，前端无需额外处理
2. **错误处理**：统一处理网络错误和业务错误
3. **加载状态**：所有异步操作都应有加载提示
4. **空状态**：对话历史为空时显示引导提示
5. **敏感信息**：不要在前端暴露API Key
