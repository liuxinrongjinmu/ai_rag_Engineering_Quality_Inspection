# CONSENSUS - 工程质检RAG系统

## 一、需求描述

构建一个公路工程质量检测RAG系统，实现：

1. **本地知识库检索**：基于11个PDF规范文档和3个Excel数据文件
2. **网络检索补充**：当本地知识库无答案时，联网搜索权威来源
3. **答案可追溯**：每个答案必须能追溯到具体的规范条文、页码、章节

## 二、验收标准

### 2.1 功能验收

| 场景 | 问题 | 预期结果 |
|------|------|---------|
| 基础规范查询 | "土方路基压实度检测频率是多少？" | 返回JTG F80-1-2017具体要求+条文号 |
| 跨文档关联 | "水泥混凝土抗压强度试件取样方法和检测频率？" | 同时引用JTG 3420-2020和检测频率表 |
| 模糊查询 | "粉煤灰怎么检？" | 返回检测项目、标准、频率、取样方法 |
| 网络检索 | "2024年新发布的公路工程标准有哪些？" | 触发网络检索，返回权威来源信息 |
| 来源验证 | 点击"查看原文" | 跳转到原始PDF位置或显示原文片段 |

### 2.2 性能验收

| 指标 | 目标值 |
|------|--------|
| 检索准确率 | ≥80% |
| 单次问答延迟 | <3秒 |
| 并发支持 | 10 QPS |

### 2.3 交付物

1. 完整后端代码（可运行、有注释、有README）
2. 数据处理脚本（PDF解析、切片、入库）
3. 向量数据库（已入库、可直接使用）
4. API接口文档
5. 测试报告（5个场景测试结果+准确率）

## 三、技术实现方案

### 3.1 技术栈

| 组件 | 技术选择 |
|------|---------|
| 后端框架 | FastAPI |
| PDF解析 | PyMuPDF |
| Excel解析 | pandas |
| 向量数据库 | ChromaDB |
| Embedding | text-embedding-3-small / bge-large-zh |
| LLM | GPT-4o / DeepSeek |
| 网络检索 | Tavily API |
| RAG框架 | LangChain |

### 3.2 架构概览

```
用户 → FastAPI → 查询编排器 → 混合检索器 → [本地检索 | 网络检索] 
                                                    ↓
                                              重排序 → RAG引擎 → LLM → 响应
```

## 五、三条底线

### 5.1 权威性底线
- 网络检索结果**永远**标注来源
- 本地知识库答案优先级高于网络检索
- 网络检索结果必须标注"非官方来源，仅供参考"

### 5.2 可解释性底线
- 每个答案必须追溯到具体规范条文、页码、章节
- 用户可查看原始文档片段
- 不确定时明确告知，不装作确定

### 5.3 质量底线
- 数据清洗完整，不假装成功
- 切片策略合理，不支离破碎
- 检索准确率≥80%
- 代码可维护，有注释
